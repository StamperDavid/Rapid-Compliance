ACTIONS:

- Click edit icon to edit lead
- Click delete icon to remove lead


2.2 CREATING A NEW LEAD
--------------------------------------------------------------------------------
Page: /workspace/[orgId]/leads/new

STEP-BY-STEP:

1. From the Leads page, click "New Lead" button (top right)
2. Fill in the form:

REQUIRED FIELDS:
- First Name
- Last Name
- Email

OPTIONAL FIELDS:
- Company
- Phone
- Lead Source (dropdown: Web, Referral, Event, API, Manual, Import)
- Lead Status (dropdown: New, Contacted, Qualified, Unqualified)
- Lead Score (0-100)

4. You'll be redirected to the lead detail page

2.3 VIEWING LEAD DETAILS
--------------------------------------------------------------------------------
Page: /workspace/[orgId]/leads/[id]

STEP-BY-STEP:

1. From Leads list, click any lead row
2. You'll see the Lead Detail page with 4 main sections:

SECTION 1: LEAD INFORMATION (Top)
- Name, email, phone
- Company, status, score
- Source and created date

SECTION 2: ACTIVITY TIMELINE (Middle Left)
- All interactions with this lead
- Emails sent/received
- Calls made
- Status changes
- Notes added
- Shows date/time and user who performed action

SECTION 3: RELATED RECORDS (Middle Right)
- Associated deals
- Related contacts
- Open tasks
- Recent conversations with AI agent

SECTION 4: ACTION BUTTONS (Top Right)
You can perform these actions:

1. "Edit Lead" - Modify lead information
2. "Convert to Deal" - Move to sales pipeline
3. "Send Email" - Compose email to lead
4. "Make Call" - Initiate phone call (Twilio)
5. "Add to Sequence" - Enroll in email drip campaign
6. "Enrich Lead" - AI-powered data enrichment (finds company info)
7. "Add Note" - Add internal note
8. "Delete Lead" - Remove from system

USING THE ENRICH FEATURE:
1. Click "Enrich Lead" button
2. AI searches for company information, social profiles, etc.
3. Wait 5-10 seconds for results
4. Data auto-populates: company name, industry, size, location
5. Review and save

CONVERTING TO DEAL:
1. Click "Convert to Deal"
2. Modal opens with deal form
3. Pre-filled with lead information
4. Add deal-specific fields:
   - Deal Name
   - Deal Value (currency)
   - Expected Close Date
   - Stage (default: Prospecting)
5. Click "Create Deal"
6. Lead status changes to "Qualified"
7. Deal appears in pipeline

6.3 TRAINING CENTER
--------------------------------------------------------------------------------
Page: /workspace/[orgId]/settings/ai-agents/training

WHAT IS TRAINING?
The Training Center allows you to improve your AI agent through scenario-based
learning. You teach the agent how to handle specific situations.

STEP-BY-STEP:

1. Click "Settings" → "AI Agents" → "Training"
2. You'll see the Training interface:

SECTION 1: TRAINING SCENARIOS
- Pre-built scenarios (common situations)
- Custom scenarios (you create)

SECTION 2: CONVERSATION SIMULATOR
- Chat interface to test agent
- You play the customer
- Agent responds
- You rate responses

CREATING A TRAINING SCENARIO:

1. Click "New Training Scenario"
2. Fill in:
   - Scenario Name (e.g., "Customer asks about pricing")
   - Customer Message (what they might say)
   - Ideal Response (how agent should respond)
   - Context/Notes
3. Click "Save Scenario"

TRAINING THE AGENT:

1. Select a scenario
2. Chat with agent as if you're the customer
3. After each agent response, rate it:
   - ✅ Good Response
   - ⚠️ Needs Improvement
   - ❌ Poor Response
4. If poor, click "Teach Better Response"
5. Enter the correct response
6. Agent learns from your feedback

DEPLOYING TRAINED AGENT:

After training:
1. Click "Create Golden Master" button
2. Name this version (e.g., "v2 - Better Pricing Responses")
3. Review changes summary
4. Click "Deploy Golden Master"
5. New version goes live
6. Previous version archived (can rollback if needed)

TIPS:
- Train with real customer scenarios
- Use actual objections you've received
- Test thoroughly before deploying
- Deploy new versions weekly as you improve

7.3 CUSTOMER MEMORY
--------------------------------------------------------------------------------
WHAT IS CUSTOMER MEMORY?

Unlike traditional chatbots that forget customers, our platform remembers
every customer across all sessions. When a customer returns (even weeks later),
the AI agent remembers:
- Previous conversations
- Products they asked about
- Budget they mentioned
- Preferences they expressed
- Support issues they had

HOW TO VIEW CUSTOMER MEMORY:

1. Open any conversation
2. Look at right panel "Customer Context"
3. You'll see "Memory Summary" section with:
   - Key facts learned about customer
   - Purchase history
   - Communication preferences
   - VIP status (if high-value customer)
   - Open support tickets

EDITING CUSTOMER MEMORY:

1. Click "Edit Memory" button
2. Add manual notes:
   - "Prefers email over phone"
   - "Budget: $5,000/month"
   - "Decision maker: CEO approval needed"
3. Click "Save"
4. AI agent will reference these facts in future conversations

2. You'll see a list of all sequences:

   - Number of Steps

CREATING A NEW SEQUENCE:

1. Click "New Sequence"
2. Name your sequence (e.g., "New Lead Nurture")
3. Add steps:

STEP 1:
- Wait Time: 0 days (immediate)
- Subject: "Thanks for your interest!"
- Body: (compose email or use template)
- Track Opens: Yes
- Track Clicks: Yes

STEP 2:
- Wait Time: 2 days
- Subject: "How can we help?"
- Body: (compose email)

STEP 3:
- Wait Time: 4 days
- Subject: "Case study you'll love"

  - Monthly on 1st

- Entity Created

   - Message

   - Headers
   - Body

F. AI AGENT ACTION
   - Trigger AI conversation
   - With context from trigger

G. WAIT/DELAY
   - Duration (minutes, hours, days)

H. CONDITIONAL BRANCH
   - If condition true → do action A
   - If condition false → do action B

I. LOOP
   - Repeat actions X times
   - Or for each item in list
